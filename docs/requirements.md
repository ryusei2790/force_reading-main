# 要件定義書 — Blog-Read-Forced

## 1. プロジェクト概要

### 背景・目的

「後で読む」と保存した記事を結局読まない問題を解決するツール。
iPhoneで気になった記事をワンタップで保存し、毎日決まった時間にランダムで1件通知することで、**強制的に記事を読む習慣**を作る。

### プロジェクト名

`Blog-Read-Forced`

---

## 2. ユーザー・利用シナリオ

**ユーザー:** 自分（個人利用）

| シナリオ | 操作 | 期待する結果 |
|---------|------|------------|
| iPhoneでブログを見つけた | 共有 → ショートカット実行 | 「登録しました」と通知が出る |
| PCで作業中（毎日12時） | 何もしない | Chromeが通知を表示する |
| 通知をクリックした | クリック | 記事がブラウザの新タブで開く |
| ChromeでブログのURLを保存したい | 拡張のアイコンをクリック | 現在のタブが登録される |
| 全記事を確認したい | スプレッドシートを開く | 一覧で視覚的に確認できる |

---

## 3. 機能要件

### 3-1. 記事の保存（登録）

- iPhone共有シートからURLを受け取り、GASへPOSTで送信できること
- Chrome拡張のポップアップから現在タブのURL・タイトルを登録できること
- 登録データはGoogleスプレッドシートに保存されること
- タイトルが取得できない場合（iPhoneのChromeなど）はURLをタイトル代わりに使うこと
- 保存される項目：タイトル、URL、登録日時、ステータス（未読/既読）、ソース（iPhone / Chrome拡張）

### 3-2. 記事の通知

- 毎日12時にChromeがバックグラウンドで未読記事を1件ランダム取得すること
- デスクトップ通知（タイトル + 「クリックして読む」）を表示すること
- 通知をクリックすると、記事URLが新しいタブで開くこと
- 未読記事がゼロの場合は通知しないこと

### 3-3. ステータス管理（将来対応）

- 記事を読んだ後、ステータスを「既読」に変更できること（現状はスプレッドシートで手動）

---

## 4. 非機能要件

| 項目 | 要件 |
|------|------|
| コスト | 無料で運用できること（Google無料枠の範囲内） |
| インフラ管理 | サーバー管理が不要であること |
| 信頼性 | GASのWebアプリURLが変わらないこと（デプロイ管理） |
| セキュリティ | GASはURLを知る者のみアクセス可（認証なし・個人利用前提） |
| 対応環境 | iPhone（iOS標準ショートカット）+ Mac Chrome |

---

## 5. 対象外（スコープ外）

- 複数ユーザー対応
- モバイルアプリ（iOS/Android）の作成
- 記事の自動既読化
- カテゴリ・タグ管理
- 通知時刻のUI設定

---

## 6. データ仕様

### スプレッドシートのカラム構成

| 列 | カラム名 | 例 | 説明 |
|----|---------|-----|------|
| A | タイトル | GASの使い方 | titleが空ならURLを代入 |
| B | URL | https://... | 記事のURL |
| C | 登録日時 | 2025/01/15 12:30 | `new Date()` で自動設定 |
| D | ステータス | 未読 | 初期値「未読」、手動で「既読」に変更 |
| E | メモ | （空白） | 将来用 |
| F | ソース | iPhone | 登録元（iPhone / Chrome拡張） |
